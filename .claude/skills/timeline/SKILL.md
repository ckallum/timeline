# timeline: Open the Web Viewer

One command, two processes: `scripts/serve-timeline.sh` brings up qmd's HTTP search server on :3001 and Vite's dev server on :5173 together. The Vite proxy wires `/api/qmd/*` to qmd so the viewer's search box works same-origin.

---

## Behavior (default path)

1. **Rebuild timeline data if stale.** Compare `mtime` on `viewer/data/timeline.json` against journal files. If any journal file is newer (or the JSON is missing), run:
   ```bash
   cd viewer && npx tsx ../scripts/build-timeline-data.ts
   ```
   This is fast (~1s) and the viewer fetches `./data/timeline.json` at boot — stale JSON is the #1 reason the viewer shows old data.

2. **Launch the stack** in the foreground:
   ```bash
   bash scripts/serve-timeline.sh
   ```
   The script self-reports:
   - `"QMD HTTP server live on :3001 (pid N, log: .claude/logs/qmd-mcp.log)"` on success, or
   - `"qmd not installed; viewer search will be unavailable..."` when qmd is absent (viewer still works, search shows "unavailable"), or
   - `"port 3001 is already in use..."` and exits 1 if a stale qmd from another project is bound (run `qmd mcp stop` or kill it).

3. **Open the browser**:
   ```bash
   open http://localhost:5173/
   ```

4. **Report** what was launched:
   - Active days in the timeline
   - Whether data was rebuilt or skipped (mtime check)
   - Whether qmd search is live (the script's own stdout is the authoritative answer)
   - URL

The user stops the stack with Ctrl-C. The script's EXIT trap sends SIGTERM to qmd, waits up to 2s for a graceful SQLite-WAL flush, then SIGKILL if needed.

## Alternative paths (only when the default won't work)

### Viewer-only, no search
Use when you explicitly don't want qmd running (e.g. on a machine where qmd isn't installed and you just want to scroll the timeline):
```bash
cd viewer && npm run dev         # :5173, live reload
cd viewer && npm run build && npm run preview   # :4173, production bundle
```
**Caveat:** `npm run preview` disables the Vite proxy, so search will silently 404 even if qmd is running. Use `npm run dev` if you want search to work.

### Production build
`npm run build` also runs `build:data` via the `prebuild` hook and copies `viewer/data/*` into `viewer/dist/data/` automatically. Use when you're shipping a static bundle somewhere.

---

## Data source

The viewer reads `viewer/data/timeline.json`, generated by `scripts/build-timeline-data.ts`. Every capture skill triggers a rebuild after writing to the day node. `vite build` copies `viewer/data/` → `viewer/dist/data/` automatically (Vite's `outDir` resolves relative to the `viewer/` package), so a fresh `npm run build` always ships current data.

---

## Gotchas

- **Never open `viewer/dist/index.html` via `file://`.** Browsers block `fetch()` against `file://` origins; the app renders blank. Always serve over HTTP.
- **Stale dist.** `npm run build:data` updates `viewer/data/timeline.json` but NOT `viewer/dist/data/`. Only `npm run build` copies fresh data into `viewer/dist/`. If the viewer shows old data in a production bundle, run a full build.
- **Markdown formatting requires the typography plugin.** `DetailPanel.tsx` uses Tailwind `prose-*` classes which are no-ops unless `@tailwindcss/typography` is listed in `tailwind.config.js` plugins.
- **Search only works in dev mode.** The Vite proxy `/api/qmd → :3001` is mounted in the dev server, not `npm run preview`. For production search, serve both viewer and qmd from the same origin or set `VITE_QMD_URL` to an absolute URL.
- **Port collision.** `serve-timeline.sh` uses `lsof` to pre-check :3001. If it's already bound by a stale qmd from another project (the shared-default-index workaround makes this plausible), the script bails loudly. Run `qmd mcp stop` or `lsof -i :3001` + `kill <pid>` to clear it.
