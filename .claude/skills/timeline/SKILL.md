# timeline: Open the Web Viewer

Build the timeline data, compile the viewer if needed, and open it in the browser.

---

## Behavior

1. **Check if `timeline/dist/index.html` exists.**
   - If not, run: `cd timeline && npm install && npm run build`

2. **Regenerate timeline data**:
   ```bash
   cd timeline && npm run build:data
   ```
   This runs `scripts/build-timeline-data.ts` which walks `journal/` and writes `timeline/data/timeline.json`. The script has an mtime-based skip, so it exits immediately if nothing changed.

3. **Open the viewer**:
   ```bash
   open timeline/dist/index.html
   ```
   On macOS, this opens in the default browser.

4. **Report** what was built:
   - Number of active days in the timeline
   - Whether a rebuild was needed or skipped (mtime check)

## Dev mode

For development (live reload):
```bash
cd timeline && npm run dev
```
Then open the URL Vite prints (usually http://localhost:5173).

## Data source

The viewer reads `timeline/data/timeline.json`, which is generated by `scripts/build-timeline-data.ts`. Every capture skill triggers a rebuild after writing to the day node.
