# timeline: Open the Web Viewer

Build the timeline data, compile the viewer if needed, and serve it over HTTP.

---

## Behavior

1. **Check if `timeline/dist/index.html` exists.**
   - If not, run: `cd timeline && npm install && npm run build`
   - Note: `npm run build` also runs `build:data` via the `prebuild` hook, and Vite copies `timeline/data/*` into `dist/data/` automatically.

2. **Rebuild if stale.** If `timeline/dist/index.html` is older than any source file in `timeline/src/` or `scripts/build-timeline-data.ts`, run `npm run build` again. Stale dist bundles are the most common reason the viewer renders blank or misformatted.

3. **Start the preview server** (required — opening `dist/index.html` directly via `file://` blocks the `fetch('./data/timeline.json')` call and the app renders blank):
   ```bash
   cd timeline && npm run preview
   ```
   Vite prints a URL like `http://localhost:4173/`. Run this in the background so the shell is free.

4. **Open the browser**:
   ```bash
   open http://localhost:4173/
   ```

5. **Report** what was built:
   - Number of active days in the timeline
   - Whether a rebuild was needed or skipped (mtime check)
   - The preview URL

## Dev mode

For live reload during development:
```bash
cd timeline && npm run dev
```
Then open the URL Vite prints (usually http://localhost:5173).

## Data source

The viewer reads `timeline/data/timeline.json`, generated by `scripts/build-timeline-data.ts`. Every capture skill triggers a rebuild after writing to the day node. `vite build` copies `timeline/data/` → `dist/data/` automatically, so a fresh `npm run build` always ships current data.

## Gotchas

- **Never open `dist/index.html` via `file://`.** Modern browsers block `fetch()` against `file://` origins; the app will render a blank page because `useTimelineData` can't load `timeline.json`. Always serve over HTTP via `npm run preview` or `npm run dev`.
- **Stale dist.** `npm run build:data` updates `timeline/data/timeline.json` but NOT `dist/data/`. Only `npm run build` (or `vite build`) copies the fresh data into `dist/`. If the viewer shows old data, run a full build.
- **Markdown formatting requires the typography plugin.** `DetailPanel.tsx` uses Tailwind `prose-*` classes which are no-ops unless `@tailwindcss/typography` is listed in `tailwind.config.js` plugins.
